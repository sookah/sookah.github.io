class GeneralDeviceManagement[source]¶

Generic methods for general device management and monitoring.

get_proc(proc_name, proc_id, pproc_id, serialize)[source]¶

Gets a process from the list of currently running ones.

Parameters:
  • proc_name (str) –

    Name of the process to look up and perform the method for.

    Must be a string unique to the name of a single process among the currently running ones if used without proc_id.

    If unset, proc_id must be set instead. If both are set, proc_id is used for the lookup and proc_name is only used for verification.

    Internally uses get_procs_by_name() and checks if the returned list only contains a single item.

  • proc_id (int) –

    Process ID (PID) of the process to look up and perform the method for.

    If unset, proc_name must be set instead. If both are set, proc_id is used for the lookup and proc_name is only used for verification.

    If using proc_id, pproc_id can be used to indicate the parent process id in order to check for possible process id reuse.

  • pproc_id (int) –

    Expected Parent Process ID (PPID) for the process returned when looking up proc_id.

    Used to check for possible process id reuse, and only in conjunction with proc_id.

  • serialize (bool) –

    Tells the method whether to return a psutil.Process instance or serialize the important process info. This is intended to separate API returns (which would be better formated with a serialized object) versus internal usage which might require a psutil.Process object.

    Defaults to False.
Returns:

Process whose attributes match the provided function arguments.
Return type:

psutil.Process or dict

static get_files_in_use()[source]¶

Get all files currently being used. To be used when general info on all files curently open on the system is required. For info on a specific process, get_proc() is recommended instead.

Returns:

Files currently open.
Return type:

list(psutil.Popen)
Example:

get_files_in_use() - list of all open files (psutil.Popen) across all processes

get_files_in_use()[0] - process name and pid, as well as all open files (psutil.Popen) for said process

get_files_in_use()[0][‘open_files’] - list of all open files (psutil.Popen) for specific process

get_files_in_use()[0][‘open_files’][0] - info about specific open file (psutil.Popen)

change_cpu_hog_priority(limit, niceness)[source]¶

Changes the priority (niceness) of processes who’ve used over a certain amount of cpu time since booting, using get_cpu_hogs() internally to identify such processes.

Niceness values range from -20 to 20 with higher values being lower priority; By default deprioritizes cpu hogs.

Parameters:
  • limit (int) – cpu time limit (since booting) above which processes are considered “hogs”. Defaults to 20.
  • niceness (int) – New process niceness to assign to processes considered “hogs”. Defaults to 10.

class SafetyChecks[source]¶

Generic methods to verify that certain device properties are within ranges that are considered safe either for testing or general usage.

safe_disk_space_limit = 95¶

Default limit to be used by safe_disk_space() if no value is passed to the function.

safe_battery_level_limit = 40¶

Default limit to be used by safe_battery_level() if no value is passed to the function.

safe_temperature_level_limit = 80¶

Default limit to be used by safe_temperature_level() if no value is passed to the function.

safe_load_avg_limit = 0.7¶

Default limit to be used by safe_load_avg_level() if no value is passed to the function.

safe_load_avg_period = 0¶

Default period to be used by safe_load_avg_level() if no value is passed to the function

safe_disk_space(limit)[source]¶

Verifies that the device’s storage space is below a safety threshold (percentage).

Parameters:limit (int) – The percentage of storage space we require free. Defaults to safe_disk_space_limit (95) if no value is passed
Returns:Indicator if the disk space is below the limit.

True if below the limit, False if above.
Return type:bool
safe_battery_level(limit)[source]¶

Verifies that the device has a sufficient battery level or is plugged in.

Parameters:limit (int) – Battery percentage required if on battery. Defaults to safe_battery_level_limit (40) if no value is passed.
Returns:Indicator whether the battery level should be a concern.

True if the device has sufficient energy, or False if the device is below the indicated battery percentage and unplugged.
Return type:bool

Warning

Always returns None on devices that are always plugged in or have no battery sensors.

safe_temperature_level(limit)[source]¶

Verifies if all device components are below a certain percentage of their factory critical temperature levels and returns any components that aren’t.

Parameters:limit (int) – Percentage of factory critical temperature levels which is considered unsafe. Defaults to safe_temperature_level_limit (80) if no value is passed.
Returns:Hardware components approaching critical temperatures.

Empty if no components pass the limit.
Return type:dict
safe_load_avg_level(limit, period)[source]¶

Verifies that the CPU load average for the specified time period is above the specified threshold

Returns True if within the safety threshold, or False if above limit.
